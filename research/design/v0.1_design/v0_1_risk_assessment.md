# V0.1 风险评估

## 风险矩阵

| 风险 | 概率 | 影响 | 应对方案 |
|------|------|------|----------|
| 微信版本升级，紧凑结构偏移变化 | 中 | 高 | 添加结构校验，失败时输出诊断信息 |
| 进程无法连接（权限不足） | 中 | 高 | 明确提示以管理员身份运行 |
| 缓存页找不到（聊天窗口未打开） | 高 | 中 | 提示先打开聊天窗口 |
| 微信进程未运行 | 低 | 高 | 提示启动微信 |
| 中文内容编码异常 | 低 | 低 | UTF-8 解码，失败降级到 ASCII |
| 解析到非消息的误匹配（假阳性） | 中 | 低 | 校验 sequence 连续性，过滤异常值 |
| 同一消息出现在多个缓存页 | 高 | 低 | 按 sequence 去重 |
| 短消息（2-3字符）大量假阳性 | 高 | 中 | 使用 7 字节前缀定位，不用内容搜索 |

## 详细风险分析

### 风险1：紧凑结构偏移变化

**概率：中。** 微信版本升级时内部结构可能调整。

**表现：** 解析出乱码、content 为空、sequence 值异常。

**应对：**
```
1. 解析后验证 sequence 是否在合理范围（>0, <1000000）
2. 验证 content 是否至少部分可解码为文本
3. 失败时输出 raw hex 供调试
4. 添加 `--offset` 参数，允许手动调整偏移
```

### 风险2：缓存页定位失败

**概率：中。** 需要聊天窗口打开，消息在缓存中。

**表现：** `search_pattern` 返回 0 结果。

**应对：**
```
1. 提示：请打开一个聊天窗口后再试
2. 提示：如果刚启动微信，先等几秒让消息加载
3. 提示：确认微信 4.1.9.56 版本
```

### 风险3：短消息假阳性

**概率：高。** 3 字节的消息在内存中有数万匹配。

**表现：** 从内容反向搜索时返回大量无效地址。

**应对：**
```
不通过内容搜索消息。
只通过前缀 `1b 02 05 09 01 01 04` 定位缓存页。
在缓存页内按 34B 步长遍历。
```

### 风险4：管理员权限

**概率：中。** 低完整性进程无法读取其他进程内存。

**表现：** `pymem` 抛出 `CouldNotOpenProcess`。

**应对：**
```
1. 捕获异常
2. 提示：请以管理员身份运行
3. 提供快捷方式：右键 → 以管理员身份运行
```

## V0.1 失败模式汇总

| 失败模式 | 用户看到 | 概率 |
|----------|----------|------|
| 正常运行，导出成功 | ✅ 正常输出 | ~60% |
| 需要管理员权限 | ❌ 错误提示 | ~20% |
| 微信未运行 | ❌ 错误提示 | ~10% |
| 未打开聊天窗口 | ⚠️ 空导出 | ~5% |
| 微信版本变化导致解析错误 | ⚠️ 乱码输出 | ~5% |
