# M17 — Unique Marker Message Test V2

> 实验时间：2026-06-06
> 方法：发送 RAY_TEST_AAA/BBB/CCC → pymem 全内存扫描 → 反向指针追踪

## 结果

| 标记 | 内容地址 | 指针表地址 |
|------|---------|-----------|
| RAY_TEST_AAA | `0x1df1c15a24c` | `0x1df24a26620` |
| RAY_TEST_BBB | `0x1df1c15a21c` | `0x1df24a26640` |
| RAY_TEST_CCC | `0x1df56f478e0` | `0x1df24a26600+0x100` |

## 内容字符串格式

每条内容字符串前有一个约 0x3c 字节的二进制前缀，然后才是 UTF-8 文本内容。

```
[0x3c byte header] [UTF-8 message content]
```

示例 (AAA):
```
+0x00: 09 7d be 81 2c ed 41 24 04 01 01 09 05 02 37 00  | 二进制头部
+0x10: 52 41 59 5f 54 45 53 54 5f 41 41 41 5f 32 30 32  | "RAY_TEST_AAA_202"
+0x20: 30 36 30 36 30 36 00                                | "6060606\0"
```

## 指针表结构

在 `0x1df24a26600` 发现一个 0x20 步长条目表：

```
+0x020: [content_ptr: 0x1df1c15a24c] [0] [???] [0]   ← AAA
+0x040: [content_ptr: 0x1df1c15a21c] [0] [0]    [0]   ← BBB
+0x100: [content_ptr: 0x1df56f478e0] ...               ← CCC
```
