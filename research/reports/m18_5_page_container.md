# M18.5 — Page Container Recovery

> 实验时间：2026-06-07
> 方法：发送 20 条 RAY_PAGE_xx 测试消息 → 反向指针追踪
> 目标会话：文件传输助手

---

## 结果

| 测试 | 结果 |
|------|------|
| 20 条消息全部在内存中找到 | ✅ 全部定位 |
| 反向指针搜索 | ❌ 0 引用（无 0x2d8 结构指针） |
| filehelper 0x2d8 块检查 | ❌ 无 PAGE 内容指针 |

## 结论

文件传输助手的消息内容存储在**紧凑结构（34B）**中，而非 0x2d8 完整消息结构体。

紧凑结构的特点：
- **内容内联**（非指针引用）→ 这就是为什么没有反向引用
- 每条 34 字节
- 约 25 条/页
- 前缀 `1b 02 05 09 01 01 04`
- 内容在 `+0x07` 偏移处

## 当前两条并行路径

```
路径 A: 紧凑结构 (34B) ← 文件传输助手使用此路径
  内容内联，可直接提取

路径 B: 0x2d8 完整结构 ← 群聊/联系人使用此路径
  +0x120 = 接收者
  +0x268 = 内容指针
  已验证（M18）
```
