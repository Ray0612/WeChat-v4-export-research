## 结论

### 判断：Case A — 本地缓存全量历史

```
Online Earliest  = 2025-06-21
Offline Earliest = 2025-06-21
Online == Offline ✅
```

**证据链：**

1. 离线翻页 143 次全部命中 GetPagedMessages
2. counter 从 4253 递减至 1，无中断
3. 离线到达的时间点与在线完全一致
4. 到达边界后不再触发函数调用（明确终止，非异常中断）
5. 无错误提示、无联网请求

### 存储结论

| 问题 | 答案 |
|------|------|
| 本地是否缓存全量历史？ | **是的**（至少 319 天/~1 年） |
| 缓存是否有时间限制？ | 本实验未发现（至少到 2025-06-21） |
| 边界是否是聊天创建时间？ | **很可能** — 无断层，平滑终止 |
| 存储位置？ | 仍未知（非标准 .db / .sqlite / .ldb） |

### 关键推论

**微信 4.1.9.56 在本地存储了完整的聊天历史**（不仅仅是最近几条）。离线状态下可以完整翻阅该聊天的全部消息记录。存储格式不是标准数据库，但消息数据一定以某种形式存在于本地文件系统中。
