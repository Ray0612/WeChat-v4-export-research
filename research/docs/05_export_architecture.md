# 微信 4.1.9.56 内存导出架构

## 已验证的缓存结构

### 消息紧凑缓存（Primary 消息数组）

**位置：** 堆内存（RW）
**元素大小：** **34 字节/条**（固定步长）
**排列方式：** **倒序**（最新消息在最低地址）
**容量：** 约 10 条/页（推测，依据 MSG_001-010 完全连续）

```
低地址                     高地址
┌────┬────┬────┬────┬────┬────┬────┬────┬────┬────┐
│010 │009 │008 │007 │006 │005 │004 │003 │002 │001 │ (MSG)
│350 │349 │348 │347 │346 │345 │344 │343 │342 │341 │ (seq)
└─34─┴─34─┴─34─┴─34─┴─34─┴─34─┴─34─┴─34─┴─34─┴─34─┘
 ← 新消息           旧消息 →
```

### 消息结构（紧凑格式，34 字节）

```
偏移  长度  字段
+0    7    prefix (固定值: 1b 02 05 09 01 01 04)
+7    var  content (消息文本，变长)
+?    1    separator (固定值 04)
+?    2    sequence (小端序 uint16，同会话+1)
+?    3    session_tag (9e 96 XX，XX 可能标识会话)
+?    8    ptr (堆指针)
+?    4    user_id (6a 22 5e XX)
+?    2    tail (81 be)
───────────────────────
总计: 34 字节（包括内容后的填充到 34）
```

---

## 导出策略

要从内存遍历消息，有两种可行路径：

### 路径A：通过紧凑结构遍历

```
已知一条消息的地址
    ↓
+34 字节 = 下一条消息（更旧）
    ↓
-34 字节 = 上一条消息（更新）
    ↓
遍历整个缓存页
    ↓
缓存页末尾 → 可能指向下一页
```

**优点：** 结构简单、步长固定、内容明文
**缺点：** 缓存深度有限（~10条/页），需要跨页寻址

### 路径B：通过序列号寻址

```
已知一条消息的序列号
    ↓
seq - 1 = 同会话上一条消息
    ↓
在内存中搜索 msg_content（已知部分）或序列号
    ↓
定位目标消息
```

## 已知的缓存区域

| 区域 | 地址范围 | 包含消息 | 步长 |
|------|----------|----------|------|
| HELLO 缓存 | 0x1f472622... | HELLO_1-5 + TEST_AAAAA | 34B |
| MSG 缓存 | 0x1f47219e... | MSG_001-010 | 34B |
| ProtoBuf 区 | 分散 | 完整消息对象 | 不固定 |
| 记录区 | 分散 | wxid/群ID/群名 | 不固定 |

---

## 推荐遍历方案

```
1. 从 Weixin.exe 内存中搜索 1b 02 05 09 01 01 04（紧凑结构前缀）
2. 找到所有紧凑结构缓存页
3. 按地址聚类（同区域=同缓存页）
4. 每页按 34 字节步长遍历
5. 提取 sequence + content + session_tag
6. 同 session_tag + 连续 sequence = 同会话消息链
7. 通过记录区中的 receiver/chatroom_id 关联到联系人
```

---

## 已知限制

1. **缓存深度有限**：一次只能遍历到 ~10 条左右的消息
2. **历史消息需翻页**：旧消息缓存可能在其他堆页，需要触发翻页才能加载到缓存中
3. **ProtoBuf 回收**：完整消息对象可能被 GC 回收，ProtoBuf 格式不如紧凑结构持久
4. **短字符串假阳性**：2-3 字符的消息在 `pattern_scan_all` 中有大量假阳性（A1 有 25110 匹配）

---

## 可行导出管道（建议）

```
Weixin.exe 进程
    ↓ (pymem. ReadProcessMemory)
紧凑结构缓存区扫描
    ↓ (34 字节步长解析)
消息对象列表 [content, sequence, timestamp, session_tag]
    ↓ (关联记录区)
会话映射 [receiver, chatroom_id, chatroom_name]
    ↓
导出到文本文件
```

**核心复杂度：O(n)，n = 当前缓存中的消息数量。**
